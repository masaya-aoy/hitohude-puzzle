# 一筆書きロジックパズル (Line Logic Puzzle)

## 🎮 アプリの概要

「一筆書き」と「数字の順序制約」を組み合わせたスマートフォン向けのロジックパズルゲームです。全マスを一度ずつ通過するルールに、独自の制約（指定順通過、障害物、〇番目以外・以下などの条件）を加えました。

👉 【ゲームをプレイする (Demo)】https://masaya-aoy.github.io/hitohude-puzzle/index.html

## ✨ 特徴と機能

初心者向けの基本編10コースと、論理的思考が求められる応用編15コースを実装

ブラウザ上で軽快に動作し、アプリのインストール不要

直感的なタッチ操作（途中で指が離れても続きから描けるスマホ特化のUX）

ミスを瞬時に知らせるリアルタイム判定と、ワンタップでやり直せるリセット機能

進行状況の自動セーブ機能（Local Storage活用）

## 🛠 技術スタック

言語: HTML5, CSS3, JavaScript (Vanilla JS)

環境: GitHub, GitHub Pages (ホスティング)

開発手法: 生成AIを活用した要件定義・プロトタイプ開発
※ライブラリやフレームワークに依存せず、ブラウザの標準機能のみで描画とパズルロジックをゼロから実装しています。

## 💡 工夫した点

【担当箇所】
企画立案、要件定義、UI/UX設計、AIを活用した開発ディレクション

【工夫した点】

1. スマホユーザーに寄り添ったUX設計
スマホでの操作を前提とし、なぞる途中で指が離れても「最後に通ったマスから続きを描ける」仕様や、画面操作時の誤スクロール防止など、プレイヤーのストレスを減らす遊びやすさを追求しました。

1. 段階的な難易度と独自のパズルロジック
単なる一筆書きではなく、「障害物」や「〇番目以下・〇番目以外で通過する」といった独自の複雑な条件（あいまい制約）を考案・実装しました。これによりパズルとしての戦略性と奥深さを高めました。

1. 生成AIを活用したアジャイル開発
プログラミング未経験ながら、生成AIを開発パートナーとして活用しました。自ら要件やルールを論理的に言語化して指示を出し、壁打ちを通じてロジックの矛盾やバグを潰しながら実装を進めることで、エンジニアと協業するための「要件定義力」と「ディレクション能力」を身につけました。

## 📱 遊び方（ルール）

画面を指でなぞって「一筆書き」でマスを繋ぎます。

数字が書かれたマスは、その順番通りに通過する必要があります。

応用編では「7x（7番目以外）」「4↓（4番目以下）」などの特殊な制約が登場します。

全てのマス（×印の障害物を除く）を青く塗りつぶせばクリアです！
